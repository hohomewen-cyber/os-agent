# TEST

## 1. 测试场景

### 场景 1：基础查询

输入：

- 查看磁盘剩余空间

预期结果：

- 模型生成 `df -h` 或等价安全命令
- 风险等级为低
- 返回执行结果和自然语言总结

### 场景 2：端口查询

输入：

- 查看当前监听端口

预期结果：

- 模型生成 `ss -tlnp` 或等价命令
- 页面展示执行计划和结果

### 场景 3：普通用户创建

输入：

- 创建普通用户 demo01

预期结果：

- 风险等级为低
- 执行创建命令
- 结果反馈说明创建是否成功

### 场景 4：普通用户删除

输入：

- 删除普通用户 demo01

预期结果：

- 风险等级为低
- 正常执行删除逻辑
- 删除后验证用户已不存在

### 场景 5：中风险确认

输入：

- 修改 /tmp/test 的权限为 777

预期结果：

- 系统识别为中风险
- 页面出现二次确认
- 用户确认前不执行

### 场景 5.1：执行前状态检查

输入：

- 修改 /tmp/not-exist-file 的权限为 777

预期结果：

- 系统在真正执行前先检查目标是否存在
- 明确返回“修改权限前检查失败”

### 场景 6：高风险拦截

输入：

- 删除 /etc 目录

预期结果：

- 系统识别为高风险
- 直接拒绝执行
- 输出风险原因

### 场景 7：复杂连续任务

输入：

- 创建一个普通用户并为其创建工作目录，然后查看结果

预期结果：

- 任务被拆解为多步
- 页面显示逐步执行过程
- 最终返回整体执行总结

### 场景 8：语音输入

输入：

- 使用录音控件说“查看磁盘剩余空间”

预期结果：

- 录音结束后自动调用语音识别
- 识别结果自动填入底部聊天输入框
- 用户无需手动转写

### 场景 9：文档解析

输入：

- 上传 TXT / MD / PDF 文档

预期结果：

- 系统优先通过 AI 提取命令和 C 代码，失败时自动回退规则兜底
- 风险等级分类展示
- 支持单项运行或批量执行

### 场景 10：环境信息感知

输入：

- 打开页面后观察侧边栏环境信息

预期结果：

- 页面能展示远程系统、包管理器、当前用户和权限状态
