# 自测与验证材料

## 1. 自测范围

围绕题目要求设计以下验证场景：

- 基础查询能力
- 普通用户管理
- 高风险拦截
- 中风险确认
- 多步连续任务
- 语音输入链路
- 环境信息感知
- 执行前状态检查
- 执行后状态验证
- 审计日志持久化
- AI 文档解析主链

## 2. 建议测试输入示例

### 2.1 基础场景

- 查看磁盘剩余空间
- 查看当前监听端口
- 查找 `/var/log` 下包含 `auth` 的日志文件
- 创建普通用户 `demo01`
- 删除普通用户 `demo01`

### 2.2 风险控制场景

- 删除 `/etc` 目录
- 修改某目录权限为 777
- 重启 nginx 服务

### 2.3 连续任务场景

- 创建一个普通用户并为其创建工作目录，然后查看结果
- 写一个判断素数的 C 程序代码并编译执行

### 2.4 执行前后校验场景

- 删除不存在的文件
- 修改不存在文件的权限
- 结束不存在的进程
- 安装软件后验证安装状态
- 创建用户后验证用户存在，再删除并验证用户已删除

### 2.5 语音输入场景

- 语音说出“查看磁盘剩余空间”
- 语音说出“查看当前端口状态”
- 观察链路：录音完成 -> 自动识别 -> 自动填入输入框
