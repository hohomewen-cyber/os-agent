# 实现与演示说明

## 1. 作品概述

本项目实现了一个基于 Streamlit 的操作系统智能代理，运行于 Windows 侧，通过 SSH 代理控制 Linux 服务器。用户可通过自然语言文本或语音描述运维需求，系统完成意图解析、风险判定、命令执行与自然语言反馈。

## 2. 当前实现形态

- Web 交互端：`app.py`
- Agent 决策与 Prompt：`agent.py`
- SSH 执行器：`executor.py`
- C 代码执行器：`modules/c_executor.py`
- Windows 本地终端桥接：`modules/windows_terminal.py`
- 文档批量解析：`modules/document_parser.py`
- 语音输入与识别：`voice_input.py`

## 3. 已实现能力

- 自然语言驱动 Linux 运维操作
- 基础系统信息查询：磁盘、内存、端口、进程等
- 文件与目录类查询
- 普通用户创建与删除
- 远程环境信息感知：系统、发行版、包管理器、当前用户、权限状态
- 高风险与敏感命令识别
- 中高风险二次确认
- 执行前状态检查与执行后状态验证
- 多步连续任务编排与执行
- 审计日志持久化、导出与手动清空
- AI 主导的文档命令 / C 代码解析，规则兜底
- 执行结果与风险依据的自然语言反馈
- 语音录音、自动识别、自动填入输入框

## 4. 语音输入演示建议脚本

1. 打开应用后直接使用页面底部上方的语音录音控件。
2. 说出示例指令，例如“查看磁盘剩余空间”。
3. 结束录音后，系统自动调用千问 ASR 进行转写。
4. 识别文本自动写入底部聊天输入框。
5. 提交后展示命令计划、风险等级、执行结果与反馈说明。

## 5. 视频录制建议覆盖点

- 文本输入基础能力
- 语音输入基础能力
- 中风险命令二次确认
- 高风险命令拦截
- 多步连续任务
- 执行结果反馈与审计信息展示

## 6. 运行方式

```bash
streamlit run app.py
```

## 7. 环境说明

- Python 3.10+
- Streamlit 1.55.0
- 需配置通义千问 API Key
- 需准备可访问的 Linux 服务器环境
